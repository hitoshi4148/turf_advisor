# AI芝生質問箱

芝管理に関するAIチャットアプリケーション。Google Gemini APIを使用して芝管理の質問に回答します。

## 機能

- 緯度経度による場所の指定
- 4つのカテゴリーから選択可能なradio button
  1. 場所タイプ：未指定/ゴルフコース/グラウンド
  2. グリーンタイプ：未指定/暖地型グリーン/寒地型グリーン
  3. 芝タイプ：未指定/暖地型/寒地型/WOS/IS
  4. 回答モード：未指定/慎重に回答
- 選択した情報を元に前文と後文を自動生成
- クッキーによる設定の保存・読み込み

## セットアップ

1. 依存パッケージのインストール
```bash
npm install
```

2. Gemini APIキーの取得
   - [Google AI Studio](https://aistudio.google.com/)にアクセス
   - Googleアカウントでログイン
   - 左側のメニューから「Get API key」をクリック
   - 「Create API key」を選択
   - プロジェクトを選択または新規作成
   - 生成されたAPIキーをコピーして安全に保存

3. 環境変数の設定
`.env.example`をコピーして`.env`ファイルを作成し、Gemini APIキーを設定してください。
```bash
copy .env.example .env
```

`.env`ファイルを編集して、`GEMINI_API_KEY`に取得したAPIキーを設定してください。
```
PORT=3000
GEMINI_API_KEY=ここに取得したAPIキーを貼り付け
```

4. サーバーの起動
```bash
npm start
```

5. ブラウザでアクセス
```
http://localhost:3000
```

## Render.comでの公開方法

### セキュリティについて

**重要**: `.env`ファイルは`.gitignore`に含まれているため、GitHubにコミットされません。APIキーは安全に管理されています。

### Render.comでのデプロイ手順

1. **GitHubリポジトリにプッシュ**
   ```bash
   git add .
   git commit -m "Initial commit"
   git push origin main
   ```

2. **Render.comで新規Webサービスを作成**
   - [Render.com](https://render.com)にログイン
   - 「New +」→「Web Service」を選択
   - GitHubリポジトリを接続

3. **環境変数の設定**
   - Render.comのダッシュボードで、作成したサービスの「Environment」タブを開く
   - 以下の環境変数を追加：
     - `GEMINI_API_KEY`: あなたのGemini APIキー
     - `PORT`: `3000`（Render.comが自動設定する場合は不要）
     - `GEMINI_MODEL`: `gemini-2.5-flash`（オプション、デフォルト値あり）

4. **ビルド設定**
   - Build Command: `npm install`
   - Start Command: `node server.js`

5. **デプロイ**
   - 「Create Web Service」をクリック
   - デプロイが完了すると、自動的にURLが発行されます

### 注意事項

- **APIキーは環境変数として設定**: `.env`ファイルではなく、Render.comの環境変数設定画面で設定してください
- **無料枠の制限**: Gemini APIの無料枠には使用制限があります。大量のリクエストがある場合は、APIキーの使用量を監視してください
- **APIキーの再生成**: もしAPIキーが漏洩した可能性がある場合は、Google AI Studioで新しいキーを生成し、古いキーを無効化してください

## 使用方法

1. 緯度・経度を入力（任意）
2. 4つのカテゴリーから適切な選択を行います
3. チャット欄に質問を入力して送信
4. 設定は自動的にクッキーに保存され、次回訪問時に自動読み込みされます

## サーバーコンソールでのログ確認

サーバーを起動したターミナル（コマンドプロンプトやPowerShell）で、送信されるプロンプトの全文を確認できます。

### 確認手順

1. **サーバーを起動**
   ```bash
   npm start
   ```
   起動すると以下のようなメッセージが表示されます：
   ```
   サーバーがポート 3000 で起動しました
   http://localhost:3000 にアクセスしてください
   ```

2. **ブラウザでチャットを送信**
   - ブラウザで `http://localhost:3000` にアクセス
   - チャット欄に質問を入力して送信

3. **ターミナルでログを確認**
   チャットを送信すると、ターミナルに以下のようなログが表示されます：
   ```
   ================================================================================
   送信プロンプト全文:
   ================================================================================
   [前文の内容]
   
   [ユーザーのメッセージ]
   
   [後文の内容]
   ================================================================================
   
   Using model: gemini-2.5-flash
   ```

### ログの内容

- **送信プロンプト全文**: Gemini APIに送信される完全なプロンプト（前文＋ユーザーメッセージ＋後文）
- **使用モデル**: 現在使用しているGeminiモデル名
- **エラー情報**: エラーが発生した場合、詳細なエラーメッセージも表示されます

### 注意点

- サーバーを起動したターミナルウィンドウを閉じると、サーバーが停止します
- ログはリアルタイムで表示されるため、チャットを送信するたびに新しいログが追加されます
